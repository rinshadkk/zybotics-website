Kernel Architecture

A Kernel Built for Intelligence

A clean hub-and-spoke architecture. Agent processes register, connect via persistent WebSockets, discover each other's capabilities, and compose into powerful automated workflows — all managed by the orchestrator kernel.

OS Capabilities

Everything a Kernel
Provides — for Agents

Scheduling, memory, IPC, governance, sandboxing. All the primitives your agents need to run reliably in production — without you building any of it from scratch.

🔄

Workflow Planning & Execution

LLM-powered planner discovers available agent capabilities, generates multi-step WorkflowPlans, and forwards to the executor — all automatically. Step outputs chain into next-step inputs via template substitution.

plan_task execute_workflow step-chaining
🧠

Cortex Long-Term Memory

Per-agent persistent memory backed by Markdown files and SQLite FTS5. Organized into typed categories (Facts, Preferences, Patterns, Instructions, Errors). Auto-compresses via LLM when categories grow beyond thresholds.

memory_write FTS5 search auto-compress
⚡

Centralized LLM Proxy

All agent LLM traffic flows through a single gateway with rate limiting, daily token budgets, cost tracking, guardrails (blocked phrases, input/output scanning), and per-agent usage analytics — visible in the dashboard.

Anthropic OpenAI Gemini rate-limit
📡

Real-time WebSocket Hub

Agents register once and maintain persistent WebSocket connections. Heartbeat monitoring (15 s ping / 30 s timeout) ensures the registry stays accurate. Agent IDs persist across restarts via .agent_id files for seamless reconnects.

FastAPI EventBus SSE
📊

Live Monitoring Dashboard

A full-featured browser dashboard: agent registry, workflow tracing with per-step live updates, LLM usage analytics with query drawer, Cortex memory browser, and push-based settings management — all over SSE.

SSE streams real-time no framework
🛡

Settings Push & Live Control

Change any configuration from the dashboard and broadcast to all connected agents instantly via settings_push WebSocket messages. Kill switches, rate limits, and security policies apply without restarts.

live-reload kill-switch zero restart

Kernel Security

Privilege Rings for AI.
Zero Compromises.

Like a hardened OS kernel, Agents Collective enforces security at every layer — from the syscall interface (LLM Proxy guardrails) to process isolation (subprocess sandboxing) to storage access control (path-sandboxed filesystem). Agents only ever get what they're allowed to have.

🔴

Kill Switch

Per-agent and global kill switches pushed live from the dashboard. Zero restart required.

🔐

Fernet-Encrypted Settings

All API keys and secrets stored encrypted via Fernet symmetric encryption in the settings store.

🛡

LLM Guardrails

Input/output phrase blocking, daily token budgets, and per-agent rate limiting on all LLM traffic.

🔒

Explicit Consent Gates

Document Agent requires per-call user_consent=true before any extracted text reaches an LLM.
